Overview:

1.    Diner Fbx Model is just TPose with Mesh

2.    Diner Prefab has Animator with mix of sitting and standing animations, as well as sit to stand and stand to sit

3.    The root for sitting animations is as captured in real life

(if it is below the seat, then it is not between the feet):

4.    Therefore a fix you can apply is to use the Root object that sits below the seat to store the offset from a foot Root, and make the Control Rig Reference (Character_Ctrl:Reference) have an offset to between the feet.

5.    Finally when you use both sets of animation in an Animator, you can select the root that is between the feet for animation settings and any adjustments.

6.    In our case it is always Root Transform in the Animations window, or Control Rig Reference (Character_Ctrl:Reference), the top root level, as the seat root offset in stored on the child Root object
